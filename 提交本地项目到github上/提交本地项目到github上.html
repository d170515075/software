提交本地项目到github上

村里有个小芳是姑娘发布于 2016-11-07

使用git客户端将本地项目上传到github上的简要步骤。

1.第一步：安装软件

安装git for window客户端，过程不再详述,一直next即可。

2.第二步：在本地创建SSH

安装完毕后，在任意目录，右键点击“git Bush” 启动 git的命令行。

输入：ssh-keygen -t rsa -C "这里填写你的github邮箱"
按4次回车。出现如下界面：

任意编辑工具打开上述文件之后，复制里面的所有内容。
如：

3.第三步：在github网站中添加ssh

进入自己github的主页面。



点击Add SSH key，再根据提示输入密码即可添加成功。你本地的客户端就可以连接geihub了。

在命令行中输入以下命令验证是否添加成功：
ssh -T git@github.com

如果出现：
You've successfully authenticated, but GitHub does not provide shell access
就代表已经可以成功连上自己的github账户了。

4.第四步：将本地的项目提交到github

提交之前，需要先配置用户名和邮箱：

右键git bush进入命令行，输入：
$ git config --global user.name "你的git用户名"
回车
$ git config --global user.email "你的git邮箱"
回车

提交项目分两种情况：

1.远程已经存在项目，我们需要先clone下来，修改代码，再提交；

命令行输入:

git clone https://github.com/用户名/项目名.git
即可将远程项目克隆下来;

克隆下来后,可以对代码进行修改操作，然后进入克隆下来的项目文件夹的命令行输入：
git add ./*
即可将当前目录下的所有文件添加到提交列表

再次输入:
git commit -m '提交信息'
将代码提交到本地的head中,此时还没有同步到远程去.

接着输入:
git push origin master
将改动的文件提交至远程的master分支下.master也可以换成其他分支
这样我们就可以在远程github网站看到改动的信息了。

2.远程没有项目，提交本地项目。

先在github网站新建立一个空白的项目,
再进入要提交的项目文件夹中，右键git bush打开命令行。
输入: git init
使项目成为一个仓库.

输入: git remote add origin git@github.com:fernandozhaxi/memeda.git
将本地项目与远程项目关联起来.

关联好了之后, 输入：
git add ./*
即可将当前目录下的所有文件添加到提交列表

再次输入:
git commit -m '提交信息'
将代码提交到本地的head中

接着输入:
git push origin master
